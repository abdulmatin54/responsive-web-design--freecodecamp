Introduction

HTML (HyperText Markup Language) is the standard markup language used to create web pages.

It provides the structure and content of a webpage through a series of elements and tags.

HTML is interpreted by browsers and works in combination with CSS and JavaScript.

It defines headings, paragraphs, images, links, and more.

HTML5 is the latest version and includes semantic elements and APIs.

<!DOCTYPE html>
<html></html>
  • Markup language
  • Core of web development
  • Works with CSS/JS
  • Uses tags and elements
  • Interpreted by browsers

HTML vs XML

HTML and XML are both markup languages but serve different purposes.

HTML is used for displaying content in browsers, while XML is for storing and transporting data.

HTML tags are predefined; XML allows users to define custom tags.

HTML is forgiving of errors; XML is strict with syntax rules.

They can be used together in web applications.

<note>
  <to>User</to>
  <from>Admin</from>
</note>
  • HTML for content
  • XML for data
  • Predefined vs custom tags
  • Loose vs strict syntax
  • Different use cases
Basic structure

HTML documents follow a basic structure that includes key elements.

The structure includes the DOCTYPE, html, head, and body tags.

The head contains metadata, title, links, etc.

The body holds all visible page content.

This layout is the foundation of every HTML page.

<!DOCTYPE html>
<html>
  <head></head>
  <body></body>
</html>
  • DOCTYPE at the top
  • <html> wraps everything
  • <head> contains metadata
  • <body> has visible content
  • Follows a clear hierarchy
Common elements

HTML includes many elements for displaying different types of content.

Some of the most used are headings, paragraphs, images, and links.

Elements can have attributes like class, id, src, and href.

Semantic elements like <article>, <section>, and <nav> improve structure.

Understanding these helps you build better, accessible webpages.

<a href="https://example.com">Visit Example</a>
  • <h1> to <h6> for headings
  • <p> for paragraphs
  • <img> for images
  • <a> for links
  • Use semantic tags for clarity
